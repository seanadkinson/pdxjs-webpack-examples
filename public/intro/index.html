Why Module Bundles?

How many web pages look like this?

<head>
    <link rel="stylesheet" href="/stylesheets/base.css" type="text/css" media="screen">
    <link rel="stylesheet" href="/stylesheets/syntax.css" type="text/css" media="screen">

    <script type="text/javascript" src="/javascripts/jquery.min.js"></script>
    <script type="text/javascript" src="/javascripts/something.that.depends.on.jquery.js"></script>
    <script type="text/javascript" src="/javascripts/another.dependency.js"></script>

    <link rel="stylesheet" href="/stylesheets/another.dependency.styles.css" type="text/css" media="screen">
</head>

This is bad for many reasons:

  • Hard to maintain
  • Implicit dependencies instead of explicit
  • Inefficient loading
  • Dev environment typically different than production

There's A Better Way

Bundle all the files together

<body>
    <script type="text/javascript" src="/javascripts/bundle.js"></script>
</body>
